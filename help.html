🌾 Page Harvester v2

Multi-page deep capture - HTML, network, timing, clicks, storage across an entire site session

How to Use

1
Log into the target siteBe authenticated before recording so cookies and auth headers are captured.
2
Click 🌾 → Start RecordingBadge goes red (REC). The debugger attaches and stays attached across page navigations.
3
Navigate the site normallyClick every button, visit every section. Each page you navigate to is captured as a separate snapshot - including load time, all network calls, and UI state.
4
Stop & ExportA full folder is downloaded - one file per page plus global index files.

Output Folder Structure

harvest_college_edu_2026-06-08/ ├── index.json ← session manifest, navigation flow, folder map ├── navigation_flow.json ← page-by-page journey with load times ├── api_catalogue.json ← every API endpoint across all pages, deduplicated ├── cookies.json ← all cookies with flags (httpOnly, sameSite, etc) ├── bot_guide.json ← LLM-ready automation guide (feed this to Claude) └── pages/ ├── page_001_college_edu_login/ │ ├── snapshot.html ← full DOM with inline styles │ ├── network.json ← all requests + full timing breakdown │ ├── dom.json ← forms, buttons, stylesheets, meta tags │ └── interactions.json ← click log, localStorage, sessionStorage ├── page_002_college_edu_registration/ │ └── ... same 4 files ... └── page_003_college_edu_courses/ └── ... same 4 files ...

Key File Details

bot_guide.json
Paste this directly into Claude. Contains: all API endpoints with sample request/response, auth headers identified, form field names, timing recommendations (e.g. "wait 743ms between calls"), cookies required.
pages/pageNNN/network.json
Every network request for that page: URL, method, headers, POST body, response body (for XHR/Fetch), and precise per-phase timing — DNS lookup, TCP connect, TLS handshake, server wait (TTFB), download. This is what powers exact latency simulation.
pages/pageNNN/snapshot.html
Full live DOM snapshot. Open in any browser for visual reference. Contains inline styles for offline viewing.
navigation_flow.json
The exact journey: URL → URL → URL with timestamps and per-page load durations. Essential for simulating the real site's pacing.

Building a Bot from the Capture

Paste bot_guide.json into Claude with something like:

"Here is a full capture of my college's course registration portal. Write a Python Playwright script that: logs in, navigates to registration, registers for CS101 and MATH201, and submits. Use the exact endpoint URLs, headers, cookies, and POST bodies from this capture. Add realistic delays based on the timing data."

Tips

Start recording before the first page loads to capture initial document requests.

Don't open Chrome DevTools while recording - it detaches the CDP debugger. Close DevTools first.

For SPAs, pause a few seconds on each "view" so the DOM snapshot captures the right state.

The recreation UI (coming soon) will take this entire folder and spin up a local mock server that replays all API responses with correct timing.

⚠️ The exported files contain session cookies and auth tokens. Treat them as sensitive - don't share or commit to git.
